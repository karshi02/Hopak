# Hopak — สรุปโปรเจกต์ (Project Brief)

<!-- > เอกสารสรุปสำหรับส่งต่อให้ AI / นักพัฒนาคนอื่นเข้าใจงานทั้งหมด -->
> Hopak = แพลตฟอร์มค้นหา จอง และจัดการหอพักออนไลน์ (ตลาดหลัก: นักศึกษา ม.มหาสารคาม และจังหวัดอื่น)

---

## 1. ภาพรวมระบบ

Hopak เป็นแพลตฟอร์มจองหอพัก มี 3 ฝั่งผู้ใช้ ใช้บัญชีเดียว (SSO) และ Backend API กลางร่วมกัน:

| แอพ | ผู้ใช้ | แพลตฟอร์ม |
|-----|--------|-----------|
| **Hopak** | ผู้เช่า (นักศึกษา/คนทำงาน) | iOS · Android · เว็บ |
| **Hopak Partner** | เจ้าของหอพัก | มือถือ + เว็บ Console |
| **Hopak Admin** | ผู้ดูแลระบบ | เว็บ Console |

---

## 2. บทบาทและความสามารถ

### 2.1 ผู้เช่า (User / Tenant)
- สมัคร/เข้าสู่ระบบ (Google หรือ อีเมล/เบอร์) + ตั้งค่าโปรไฟล์
- ค้นหาหอพักตามจังหวัด/พิกัด/มหาวิทยาลัย
- ดูรายละเอียดหอ: รูป, ราคา/เดือน, ค่าน้ำ-ไฟ, สิ่งอำนวยความสะดวก, สถานที่ใกล้เคียง, พิกัด GPS
- แจ้งเตือน & โปรโมชัน (ไอคอนกระดิ่ง) — โปรจากหอพัก + สถานะการจอง
- คาโรเซลสปอนเซอร์ (หอพักที่จ่ายค่าโฆษณา) ในหน้าแรก

**Flow การจอง (สำคัญ — ลำดับตายตัว):**
1. ส่งคำขอจอง (กรอกชื่อ + เบอร์โทร + หมายเหตุ) — *ยังไม่ชำระเงิน*
2. สถานะ "รอเจ้าของหอยืนยัน" (timeline)
3. เจ้าของหอกดยืนยัน → สถานะเปลี่ยนเป็น "ยืนยันแล้ว"
4. ผู้เช่าชำระเงิน (โอนเข้าบัญชีส่วนกลาง Hopak, ตัดยอดภายในเที่ยงคืน)
5. ระบบออกใบจอง + ใบเสร็จ (PDF)
- **ยกเลิกได้ภายใน 1 วันเท่านั้น** หลังจากนั้นปุ่มยกเลิกถูกล็อก

### 2.2 เจ้าของหอพัก (Owner / Partner)
- สมัครเปิดหอ (ผ่านปุ่ม "เปิดหอพักกับ Hopak" ในแอพผู้เช่า → SSO บัญชีเดิม)
- **เพิ่ม/แก้ไขหอพักเองได้ทันที ไม่ต้องรอแอดมิน** (ยกเว้นหอใหม่ที่รออนุมัติครั้งแรก):
  - อัปโหลดรูป, เขียนคำอธิบายเอง, ปักหมุดพิกัดบนแผนที่
  - ตั้งราคา/ประเภทห้อง (แอร์/พัดลม), ค่าน้ำ-ไฟ, มัดจำ, สิ่งอำนวยความสะดวก
- จัดการห้อง: เพิ่มห้องว่าง, ตัดห้องเมื่อเต็ม (auto), เปิดห้องว่างใหม่
- รับคำขอจอง → กดยืนยัน → ระบบออกใบจองอัตโนมัติ
- ดูใบจองรายวัน + ปริ้น PDF
- แดชบอร์ด: รายได้, ยอดจอง, ห้องว่าง, กราฟรายเดือน
- ตั้งค่าบัญชีรับเงิน + วิธีชำระเงิน (รับยอดหลังหักค่าบริการ 10%)

### 2.3 แอดมิน (Admin)
- แดชบอร์ด: รายได้ระบบ, กราฟรายเดือน/ปี, ยอดจองต่อจังหวัด
- การจอง & สถานะ: ทุกการจอง, กรองสถานะ, Export
- หอพัก (อนุมัติ): คิวรออนุมัติ — ดูเอกสารแนบ + พิกัด + เบอร์ → **รับ/ไม่รับ**
- ผู้ใช้ (Users): จัดการผู้เช่า/เจ้าของหอ, ระงับ, เพิ่มผู้ใช้
- การเงิน & รวมบิล: หัก 10%, โอนให้หอพัก, ออกใบเสร็จ/รวมบิล PDF
- โฆษณา & แคมเปญ: Boost, แบนเนอร์, Featured, โปรโมชัน
- ผู้ดูแล (Admins): Super Admin / Admin / Finance / Support (แบ่งสิทธิ์)
- เปลี่ยนธีมระบบ

## 3. โมเดลรายได้ (รายได้จากไหน)

1. **ค่าคอมมิชชัน 10%** (หลัก) — หักจากทุกการจองที่สำเร็จผ่านระบบ
2. **ฟีดโปรโมท (Boost)** — เจ้าของหอจ่ายเพื่อดันหอขึ้นฟีด
3. **แพ็คโฆษณา** — แบนเนอร์ในแอพ/เว็บ
4. **แพ็คโปรโมชัน** — ป้าย Featured / แนะนำ
5. **ค่าบริการเพิ่มเติม** — ฟีเจอร์เสริม
6. *(เฟสถัดไป)* คะแนนความน่าเชื่อถือของหอ (GPA) ใช้จัดอันดับ

## 4. เส้นทางการเงิน (Money Flow)

```
ผู้เช่าชำระเงิน (มัดจำ + ล่วงหน้า)
   ↓
บัญชีส่วนกลาง Hopak (ตัดยอดภายในเที่ยงคืน)
   ↓
หักค่าบริการ 10% = รายได้ Hopak
   ↓
โอนส่วนที่เหลือให้เจ้าของหอ + ออกใบยืนยันการชำระเงิน
```

## 5. ความเป็นส่วนตัว / กันการติดต่อนอกระบบ

- เบอร์โทรผู้จองแสดงแบบ **ซ่อนบางส่วน** ให้เจ้าของหอเห็น เช่น
  ส่งจริง `089-123-4567` → เจ้าของหอเห็น `นายสมชาย ล. · 089-123-**-*`
- เปิดเผยเบอร์เต็ม **หลังชำระเงินและออกใบจองเรียบร้อยแล้วเท่านั้น**

## 6. สถาปัตยกรรม (แนะนำ)

**หลักการ: API-first** — ออกแบบ API + DB ให้จบก่อน → เว็บเรียก API → แอพเรียก API ตัวเดิม
- เว็บและแอพเป็นแค่ "หน้าจอ" ยิงเข้า Backend API กลางตัวเดียว (`api.hopak.com`)
- ฐานข้อมูลเดียว = แหล่งความจริงเดียว (Single Source of Truth)
- Auth ด้วย JWT + SSO (บัญชีเดียวใช้ได้ทั้งผู้เช่าและเจ้าของหอ)
- Realtime ด้วย WebSocket (คำขอจองเด้งทันที) หรือ polling

### Tech Stack แนะนำ (TypeScript ทั้งระบบ)

| ชั้น | เทคโนโลยี |
|------|-----------|
| เว็บ | Next.js (React) + TypeScript + Tailwind CSS |
| แอพมือถือ | React Native (Expo) + TypeScript — โค้ดชุดเดียว → iOS + Android |
| Backend / Server | Node.js + NestJS + TypeScript |
| API | REST (หรือ GraphQL) + JWT + WebSocket |
| ORM | Prisma |
| Database | PostgreSQL (หลัก) |
| Cache/คิว | Redis |
| Storage | S3 / Object Storage (รูป, เอกสาร, PDF) |
| Payment | Payment Gateway (Omise / 2C2P / Bank API) + PromptPay |
| Notification | FCM (Android) + APNs (iOS) / Expo Notifications |

### โครงสร้าง Monorepo (pnpm workspaces + Turborepo)
```
hopak/
├─ apps/web        # Next.js
├─ apps/mobile     # React Native (Expo)
├─ apps/api        # NestJS server
└─ packages/shared # type, logic, สูตรหัก 10% (ใช้ร่วมทุกฝั่ง)
```

### Deploy
- เว็บ → Vercel · API → Railway/AWS/Docker · แอพ → App Store + Google Play (EAS Build) · DB → Supabase/AWS RDS

## 7. โมเดลข้อมูลหลัก (Data Model)

- **User** — role: `tenant` | `owner` | `admin`, ชื่อ, เบอร์, อีเมล, รูป
- **Dorm** — ชื่อ, คำอธิบาย, ownerId, จังหวัด, lat/lng, ค่าน้ำ/ไฟ, สิ่งอำนวยฯ, สถานะอนุมัติ
- **Room** — dormId, ประเภท (แอร์/พัดลม), ราคา/เดือน, สถานะ (ว่าง/ไม่ว่าง)
- **Booking** — tenantId, roomId, วันเข้าอยู่, ยอดเงิน, status: `pending` | `confirmed` | `paid` | `cancelled` | `completed`
- **Payment** — bookingId, ยอด, ค่าคอม 10%, ยอดโอนให้หอ, สถานะ, วิธีชำระ
- **Review** *(เฟสถัดไป)* — dormId, tenantId, คะแนน, ความเห็น

## 8. กฎธุรกิจสำคัญ (Business Rules)

1. การจองต้องผ่านลำดับ: ส่งคำขอ → เจ้าของยืนยัน → ชำระเงิน → ออกใบจอง (ห้ามข้าม)
2. ยกเลิกฟรีได้ภายใน 1 วันหลังจอง หลังจากนั้นยกเลิกไม่ได้
3. หักค่าบริการ 10% จากทุกยอดที่ชำระผ่านระบบ
4. เบอร์ผู้จองซ่อนบางส่วนจนกว่าจะชำระเงินเสร็จ
5. เจ้าของหอแก้ไขข้อมูลหอ/ห้อง/คำอธิบายเองได้ทันที (หอใหม่ต้องรอแอดมินอนุมัติครั้งแรก)
6. ชำระเงินโอนเข้าบัญชีส่วนกลาง ตัดยอดภายในเที่ยงคืน

## 9. แผนการพัฒนา (แนะนำ)

1. **เว็บก่อน** (เร็ว/ถูก/แก้ไว, ไม่ต้องรอ App Store review) + Backend API + Admin
2. **Partner** (เว็บ Console ก่อน)
3. **แอพมือถือ** (iOS/Android) เมื่อมีผู้ใช้จริง — เพิ่ม Push + offline

**ลำดับสร้างโค้ด (3 Phase):**
- Phase 1: Monorepo + Backend API (NestJS + Prisma + PostgreSQL)
- Phase 2: เว็บ (Next.js) เรียก API
- Phase 3: แอพ (Expo React Native) เรียก API เดียวกัน
