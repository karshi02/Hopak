[แก้แล้ว] ไม่มีการเช็ค/hash password จริง — ช่องโหว่ความปลอดภัยร้ายแรง
--------------------------------------------------------
ไฟล์: apps/api/prisma/schema.prisma, apps/api/src/modules/auth/auth.service.ts,
apps/api/src/modules/users/users.service.ts,
apps/api/src/modules/admin/admins/admins.service.ts,
apps/api/src/modules/admin/approvals/approvals.service.ts,
apps/api/prisma/seed.ts
อาการ: User model ใน Prisma ไม่มี field password เลย login() หา user จาก
email/phone อย่างเดียว ไม่เช็ค password ที่ส่งมา — ใครก็ login ด้วย email
ไหนก็ได้โดยไม่ต้องรู้รหัสผ่านจริง
แก้ไข:
- เพิ่ม password String? ใน User model + migrate (add-user-password)
- register() hash ด้วย bcryptjs (10 rounds) ก่อนเก็บ, login() เทียบด้วย
  bcrypt.compare จริง, ปฏิเสธถ้า user ไม่มี password (เช่น Google-only account)
- ใช้ bcryptjs แทน bcrypt (native) เลี่ยงปัญหา compile บน Windows
- ระหว่างแก้เจอเพิ่ม: user object (รวม password hash) ถูกส่งกลับตรงๆ ใน
  response หลายจุด (auth.service register/login, users.service findById/
  updateProfile/becomeOwner, admins.service list ใช้ include:{user:true},
  approvals.service listPending ใช้ include:{owner:true}) — แก้ให้ใช้
  select เฉพาะ field ที่ปลอดภัย/omitPassword() ทุกจุด ไม่ให้ hash รั่วออกไป
- seed.ts: ตั้ง password ทดสอบ "hopak1234" ให้ account seed (admin/owner)
สถานะ: แก้แล้ว + ทดสอบผ่านครบ 5 เคส (16/07/2026): register ไม่รั่ว password,
login รหัสถูก 201, รหัสผิด 401, account เก่าที่ไม่มี password ถูกล็อก 401
(ถูกต้อง — ต้อง reset ผ่านช่องทางอื่น), seed admin login ด้วยรหัสใหม่ 201
